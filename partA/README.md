1.Мэдээллийн далдлалт зөрчил — db_conn, users_arr public байна. Доторх хэрэгжилтийн дэлгэрэнгүй мэдээлэл гадагш гарч байна.
2.Boolean flag abuse — do_user_op(flag: number). Нэг функц дөрвөн өөр үйлдэл хийж байна. 4 тусдаа метод байх ёстой.
3.Нэршлийн алдаа — usr_mgr, get_u, find нь утгагүй товчлол. UserManager, getUserById, searchUsers байх ёстой.
4.Ойлголтын хэт их жин — get_u(id_or_email, flag) нь нэг параметрт хоёр өөр утга хүлээж авна. Тусдаа метод байх ёстой.
5.Алдааны буруу буцаалт — 'ERR_404' string буцаана. Exception throw хийх ёстой байтал string-ээр алдаа дамжуулж байна.
6.Type safety алдагдсан — any хэт их ашигласан. Тодорхой User интерфейс тодорхойлох ёстой.

## А.2 — Cache Library

### Архитектурын шийдвэр

**Factory паттерн** ашигласан шалтгаан:
- Хэрэглэгч `LRUCache`, `LFUCache`, `TTLCache` классуудыг шууд мэдэх шаардлагагүй
- Зөвхөн `Cache` интерфейс болон `CacheFactory`-г л мэдвэл хангалттай
- Ирээдүйд шинэ хэрэгжилт нэмэхэд хэрэглэгчийн код өөрчлөгдөхгүй
